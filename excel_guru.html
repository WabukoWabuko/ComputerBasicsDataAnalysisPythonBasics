1Baby-to-guru ladder

A spreadsheet is a big grid notebook. Each box holds a number or a word.

Builder explanation

Tables organize data into columns with names. Formulas use those names safely.

3Lookup architecture

Modern lookup approach

Baby: Find a student name by looking at their ID card.

Builder: Use XLOOKUP to match IDs safely.

=XLOOKUP(A2, Customers[ID], Customers[Status], "Not found")

Explain why column insertions break VLOOKUP and how XLOOKUP avoids it.

Two-way lookup

Baby: Find the box where row and column meet.

Builder: Use INDEX/MATCH to find the intersection.

=INDEX(Metrics, MATCH(A2, Regions, 0), MATCH(B2, Months, 0))

5Dynamic arrays

Baby: One formula fills many boxes.

Builder: Use FILTER and SORT to automatically update results.

=SORT(FILTER(Sales[Amount], Sales[Region]=F2), 1, -1)

Spill range design

Leave space for spill results and reference the spill with #.

Error handling

Use IFERROR or IFNA to keep dashboards clean.

9Misconceptions to correct
  • “VLOOKUP is best.” It is fragile; XLOOKUP is safer.
  • “Manual cleaning is fine.” It breaks with new data.
  • “Charts are decoration.” They are decision tools.
