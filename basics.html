1Baby-to-guru ladder
Baby explanation

The computer is like a kitchen. The CPU is the chef, RAM is the counter where the chef works, and the storage is the pantry where food is kept.

Builder explanation

The CPU executes instructions, RAM holds active data, and storage keeps data long-term. When RAM fills up, the OS uses disk swap, which is slower.

Guru explanation

Performance depends on working set size, cache locality, and IO throughput. Diagnose by observing CPU utilization, memory pressure, and disk queue length.

2Core mental models

Kernel vs user space

Baby: The kernel is the “boss” that tells apps when they can use the computer.

Builder: Apps run in user space and ask the kernel for access via system calls.

Guru: Kernel scheduling, memory protection, and driver control create stability.

  • Process = running program with its own memory space
  • Thread = execution unit inside a process
  • Context switch = CPU shifts between processes

Storage hierarchy

Baby: Pantry (storage) is slow, counter (RAM) is fast.

Builder: SSDs are faster than HDDs but cost more per GB.

Guru: Choose NVMe for high IOPS, and plan backups for durability.

  • Cache/RAM for speed
  • SSD/NVMe for low latency
  • HDD for large archives
3Real-world relationships

Why the computer “freezes”

Baby: Too many toys on a small table.

Builder: RAM is full, the OS moves data to disk, everything slows down.

Guru: Identify memory pressure and reduce working set or add RAM.

Why websites feel slow

Baby: The message is traveling a long road.

Builder: DNS + TCP + TLS adds time, plus server delay.

Guru: Measure latency, server response, and render blocking assets.

4CLI practice

Teach learners to automate repetitive tasks using a clear command sequence.

cd Desktop\Project_X mkdir assets scripts docs Get-ChildItem -Recurse Get-Content .\docs\notes.txt
Instructor tip: After each command, ask learners to explain what changed.
5Networking essentials

Request lifecycle

Baby: You ask a librarian for a book.

Builder: DNS resolves domain, TCP connects, TLS encrypts, HTTP delivers.

Guru: Diagnose by inspecting headers, round-trip time, and status codes.

Debugging checklist

  • Can you resolve the domain?
  • Is port 443 reachable?
  • Are you blocked by a proxy or firewall?
  • Is the server returning 4xx or 5xx?
