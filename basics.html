1Baby-to-guru ladder
Baby explanation

The computer is like a kitchen. The CPU is the chef, RAM is the counter, and storage is the pantry.

Builder explanation

The CPU executes instructions, RAM holds active data, storage keeps data long-term. The OS manages everything.

Guru explanation

Performance depends on working set size, cache locality, IO throughput, and scheduling policies.

3Core concepts with examples

CPU, cores, and threads

Baby: One chef can cook one thing at a time. More chefs = more tasks at once.

Builder: Cores run tasks in parallel. Threads are the tasks themselves.

Guru: Context switching costs time. CPU-bound vs IO-bound tasks require different tuning.

Memory and storage

Baby: The counter is small but fast; the pantry is big but slower.

Builder: RAM is volatile; SSD is persistent.

Guru: Memory pressure triggers paging. NVMe reduces IO latency.

Processes and scheduling

Baby: The boss gives each worker a turn.

Builder: The OS gives time slices to each process.

Guru: Priority, nice values, and CPU affinity affect performance.

Filesystems and permissions

Baby: Some files are yours, some are locked.

Builder: Permissions define read/write/execute for user, group, others.

Guru: Understand ACLs, journaling, and backup strategies.

4Networking basics

Request lifecycle

Baby: You ask a librarian for a book.

Builder: DNS resolves domain, TCP connects, TLS encrypts, HTTP delivers.

Guru: Diagnose by inspecting headers, round-trip time, and status codes.

Common network problems

  • DNS misconfiguration
  • Firewall or proxy blocking
  • High latency or packet loss
  • Server overload (5xx errors)
Real-world connection: Explain how slow Wi-Fi, bad DNS, or a busy server can make an app “feel broken.”

6Less common but useful concepts

BIOS/UEFI

Startup firmware that initializes hardware before the OS loads.

Virtualization

Run multiple virtual computers on one machine for testing or isolation.

File system types

NTFS, exFAT, ext4 — each has different trade-offs for speed and reliability.

Device drivers

Translation layer between hardware and the OS.

7Real-world applications

System troubleshooting

Identify CPU, RAM, or disk bottlenecks and recommend fixes.

IT support workflows

Document issues, reproduce errors, and communicate resolution steps.

Performance tuning

Measure baseline performance, then optimize.

8Misconceptions to correct
  • “More RAM always means faster.” Sometimes storage or CPU is the bottleneck.
  • “The internet is a single place.” It is a network of networks and services.
  • “Files are magic.” Files live in folders, and permissions control access.
